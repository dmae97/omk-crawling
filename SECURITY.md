# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.x     | ✅        |
| < 2.0   | ❌        |

## Reporting a Vulnerability

If you discover a security vulnerability in omk-crawling, please report it
responsibly:

1. **Do not** open a public GitHub issue.
2. Email the maintainers or use GitHub's private vulnerability reporting
   feature: [Report a vulnerability](https://github.com/dmae97/omk-crawling/security/advisories/new)
3. Include: description, reproduction steps, affected versions, and suggested fix.

We aim to acknowledge reports within 48 hours and provide a fix within 7 days.

## Scope

This project includes TLS fingerprint impersonation and anti-bot bypass
capabilities. These are intended for legitimate security research, authorized
testing, and data extraction within legal boundaries. Users are responsible
for complying with applicable laws and target site Terms of Service.
